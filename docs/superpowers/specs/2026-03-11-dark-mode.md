## Overview

Add a dark/light mode toggle to the main app. Light mode remains the default. Dark mode uses the same navy + orange palette as the landing page redesign. The implementation is a CSS token swap ŌĆö no component rewrites needed. A ­¤īÖ/ŌśĆ’ĖÅ button in the header persists the user's preference to `localStorage`.

---

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Mechanism | `[data-theme="dark"]` on `<html>` | CSS-only token swap, zero React re-renders |
| Default | Light mode | Existing users unaffected |
| Persistence | `localStorage('theme')` | Simple, no server round-trip |
| Anti-flash | Inline `<script>` as first child of `<html>` before `<body>` | App Router pattern ŌĆö no explicit `<head>` tag; runs before paint |
| Toggle location | Header right side, immediately before `<UserButton />` | Always accessible, consistent across all pages |
| Transition | `background-color 0.2s, color 0.2s` scoped to `.theme-ready body` | Avoids animating initial page load |
| ThemeToggle icon | Renders `null` until mounted | Prevents SSR mismatch / icon flash |

## Component Design: ThemeToggle.tsx

```tsx
'use client';
import { useEffect, useState } from 'react';

export default function ThemeToggle() {
  const [mounted, setMounted] = useState(false);
  const [dark, setDark] = useState(false);

  useEffect(() => {
    const saved = localStorage.getItem('theme');
    const isDark = saved === 'dark';
    setDark(isDark);
    setMounted(true);
    // Enable transitions now that the initial theme is applied
    document.documentElement.classList.add('theme-ready');
  }, []);

  function toggle() {
    const next = !dark;
    setDark(next);
    const value = next ? 'dark' : 'light';
    localStorage.setItem('theme', value);
    document.documentElement.setAttribute('data-theme', value);
  }

  // Render nothing until mounted to avoid SSR icon mismatch
  if (!mounted) return <div style={{ width: 32, height: 32 }} />;

  return (
    <button onClick={toggle} className="theme-toggle-btn" aria-label="Toggle dark mode">
      {dark ? 'ŌśĆ’ĖÅ' : '­¤īÖ'}
    </button>
  );
}
```

## Anti-Flash Script + Transition

### layout.tsx

The `<script>` goes as the **first child of `<html>`**, before `<body>`, using Next.js App Router pattern:

```tsx
<html lang="en">
  <script dangerouslySetInnerHTML={{ __html: `(function(){var t=localStorage.getItem('theme');if(t==='dark')document.documentElement.setAttribute('data-theme','dark');})();` }} />
  <body>
    {/* ... */}
  </body>
</html>
```

### globals.css ŌĆö transition scoped to `.theme-ready`

The `ThemeToggle` component adds `.theme-ready` to `<html>` on mount. This means transitions only activate after the first render ŌĆö preventing animation on initial page load:

```css
/* Only animate theme changes initiated by the user, not initial load */
.theme-ready body {
  transition: background-color 0.2s ease, color 0.2s ease;
}
```

## Per-Page CSS Strategy

Each page CSS file gets a `[data-theme="dark"]` block appended at the bottom targeting hardcoded hex values not covered by token inheritance. The `--d-*` overrides handle ~80% automatically.

Common patterns per page:
- `background: #fff` or `background: #ffffff` ŌåÆ `background: var(--d-surface)`
- `background: #f3f3f3` or `background: #f7f7f7` ŌåÆ `background: var(--d-bg)`
- `color: #6b7280` or `color: #9b9ba3` ŌåÆ `color: var(--d-text-mid)`
- `border-color: #e5e5e5` ŌåÆ `border-color: var(--d-border)`
- `color: #242428` or `color: #111827` ŌåÆ `color: var(--d-text)`
